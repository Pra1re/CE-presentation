Core focus 1

What is QKD?

Quantum Key Distribution is a method for two parties, usually called Alice and Bob, to create a shared secret key by sending quantum states and then using a public classical channel for checking and correction.

  • • It is used for key exchange, not for sending the actual message.
  • • Security comes from quantum measurement disturbance and the no-cloning idea.
  • • If Eve intercepts the quantum states, extra errors appear and Alice and Bob can detect it.

How QKD works in 5 steps

1
Prepare
Alice encodes bits in photon states using chosen bases.
2
Measure
Bob randomly selects measurement bases for each photon.
3
Sift
They keep only positions where their bases match.
4
Check
They compare a sample to estimate the quantum bit error rate.
5
Finalize
Error correction and privacy amplification produce the secret key.
Core focus 2

Major QKD protocols

Different protocols use different physical ideas, but all aim to generate a secret key while revealing interference from an eavesdropper.

Protocol comparison

BB84

The classic 1984 prepare-and-measure protocol. Alice sends non-orthogonal states and Bob measures with random bases.

E91

Uses entangled photon pairs. Security is linked to quantum correlations and Bell-type inequalities.

Decoy-state

Adds extra intensity levels to better handle practical weak laser sources and improve security analysis.

Why BB84 became the foundation

  • • Proposed by Charles Bennett and Gilles Brassard in 1984.
  • • Uses two different polarization bases to encode bits.
  • • Non-orthogonal states cannot be perfectly distinguished or copied.
  • • After basis sifting, Alice and Bob keep matching results as the raw key.

Key idea
If Eve measures in the wrong basis, she changes the state and increases the error rate.

Strengths and limitations

Strengths

  • • Security is tied to physical laws, not only mathematical hardness.
  • • Eavesdropping creates detectable disturbance.
  • • Useful for very high-security key exchange.

Limitations

  • • Works only for key distribution, not direct large data transfer.
  • • Needs special optics, detectors, and stable channels.
  • • Real systems can still suffer from implementation flaws.
Core focus 3

Challenges and real-world uses

Practical QKD must overcome distance, device, and cost constraints before it can scale into large communication networks.

Main technical challenges

Distance limitation and photon lossVery high
Hardware cost and alignmentHigh
Detector and source imperfectionsHigh
Large-scale network deploymentMedium to high

Quantum signals weaken strongly with distance and cannot be simply amplified like ordinary optical communication signals.

Applications and future direction

Finance & Government

Used where key compromise is unacceptable and communication confidentiality has long-term value.

Satellite QKD

Promising for extending secure key exchange beyond the distance limits of terrestrial fiber links.

Hybrid security

Many experts expect QKD to be combined with post-quantum cryptography in future secure architectures.
